Device Masker Android 16 verifier hardening and public validation package

Summary

- Hardened Android 16 emulator validation for Device Masker hooks and verifier evidence.

- Consolidated agent rules for Mobile MCP/manual UI work and validation-folder hygiene into the canonical project rules.

Code changes

- Updated the verifier target package and manifest so it can be installed/scoped as the canonical local validation target.
- Expanded verifier runtime evidence capture in `verifier/src/main/kotlin/com/astrixforge/devicemasker/verifier/VerifierActivity.kt`.
- Added verifier probes for Android ID, telephony IDs, SIM/network values, Wi-Fi, Bluetooth, Advertising ID, GSF ID, Media DRM ID, device build fields, locale/timezone, location, sensors, and WebView user agents.
- Added local verifier stubs under `verifier/src/main/kotlin/com/google/` so the verifier can exercise Advertising ID and GSF-style call paths without adding production app dependencies.
- Updated `xposed/src/main/kotlin/com/astrixforge/devicemasker/xposed/hooker/DeviceHooker.kt` so restricted identifiers return configured values before unsafe platform calls can fail or return originals.
- Updated `xposed/src/main/kotlin/com/astrixforge/devicemasker/xposed/hooker/SensorHooker.kt` to normalize sensor names for deterministic verifier checks.
- Updated `xposed/src/main/kotlin/com/astrixforge/devicemasker/xposed/hooker/WebViewHooker.kt` to spoof both static default WebView UA and instance `WebSettings.getUserAgentString()` by discovering concrete WebView provider settings classes through `WebView.getSettings()`.
- Updated `xposed/src/main/kotlin/com/astrixforge/devicemasker/xposed/hooker/LocationHooker.kt` to keep direct latitude/longitude spoofing deterministic and handle null last-known location objects safely.
- Updated `xposed/src/test/kotlin/com/astrixforge/devicemasker/xposed/hooker/DeviceProfileHookCoverageTest.kt` for the hook coverage changes.
