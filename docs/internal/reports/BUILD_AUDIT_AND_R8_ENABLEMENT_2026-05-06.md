## Changes Made

### 1. R8 Enabled in Release Build

**File:** `app/build.gradle.kts`

Changed release build type from `isMinifyEnabled = false` to `isMinifyEnabled = true` and `isShrinkResources = true`. R8 was previously disabled due to `AbstractMethodError` crashes in libxposed hooker lambdas.

**Root cause:** R8 full mode strips the `XposedInterface.Hooker` SAM interface from Kotlin lambda classes generated by `xi.hook(m).intercept { chain -> ... }`. LSPosed native code calls `Hooker.intercept(Chain)` at runtime ‚Äî if the interface is stripped, `AbstractMethodError` occurs.

**Fix:** Added ProGuard keep rules for the libxposed API hook interface hierarchy.

**Result:**
| Variant | Before | After |
|---------|--------|-------|
| debug | 25.47 MB | 25.47 MB (unchanged) |
| release | 17.96 MB | **3.81 MB** |
| ciRelease | 3.81 MB | 3.81 MB (unchanged) |

### 2. ProGuard Rules ‚Äî libxposed Lambda Protection

**File:** `xposed/consumer-rules.pro`

Added rules to protect libxposed API interfaces from R8 stripping:
- `-keep interface io.github.libxposed.api.XposedInterface$Hooker { *; }`
- `-keep class * implements io.github.libxposed.api.XposedInterface$Hooker { *; }`
- `-keep interface io.github.libxposed.api.XposedInterface$Chain { *; }`
- `-keep interface io.github.libxposed.api.XposedInterface$HookBuilder { *; }`
- `-keep interface io.github.libxposed.api.XposedInterface$HookHandle { *; }`
- `-adaptresourcefilecontents META-INF/xposed/java_init.list`

**File:** `app/proguard-rules.pro`

Added matching rules for belt-and-suspenders protection.
